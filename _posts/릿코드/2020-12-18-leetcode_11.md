주어진 크기가 같은 배열 A,B,C,D 대해서, #(a+b+c+d = 0 for a,b,c,d in A,B,C,D)를 구하라.

## 4sum II

![4sum II](/images/4sum%20II.png)

생각보다 딕셔너리의 성능이 강력하다는 것을 알게 된 문제였다. 처음엔 굉장히 naive하게 A+B와, C+D를 n^n으로 곱한 list를 만든 다음,

```python
for i in A:
    for j in B:
        newlist.append(i+j)
```

그것의 unique element를 구하고, unique element끼리 더했을 때 0이 되는 element를 찾은 다음 그것들의 숫자를 더해서 0의 개수를 찾는 방식으로 하였다.

하지만 복잡도를 보니 일일히 한개의 element에 대해서 O(n)을 쓰는것이 굉장히 time_ineffective 한 일이라는 것을 알게되어, 그냥 처음부터 dictionary를 만들어, element마다 들어있는 개수를 구한 다음에 계속해서 A+B는 몇개인지, C+D는 유니크 몇개인지 찾아서 문제를 푸는 방식으로 변경하였다.

discussion을 보다보니 진짜 신기하게 푼 사람이 있어 소개한다

```python
class Solution:
    def fourSumCount(self, A: List[int], B: List[int], C: List[int], D: List[int]) -> int:
        AB = collections.Counter(a+b for a in A for b in B)
        
        return sum(AB[-c-d] for c in C for d in D)        
```

점심나가서먹을것같애!!!!

Counter는 없는 key를 찾았을 때 keyerror를 뱉지 않고 0을 준다는 것에서 착안하여 만든 풀이이다.
