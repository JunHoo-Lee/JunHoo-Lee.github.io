파이선 내의 builtin function sort에 대해서 확인할 수 있는 문제이다.

## squares of sorted array

![squares of sorted array](/images/squares%20of%20sorted%20array.png)

풀이는 간단하다, elementwise하게 제곱해 준 다음, 그냥 sort해주면 되는데,

```python
class Solution:
    def sortedSquares(self, nums: List[int]) -> List[int]:
    
        return sorted([i*i for i in nums])
```

나는 이 답안을 내고 나서 기존에 들어왔던 배열이 ascending order로 들어왔단 점을 exploit하기 위해 마이너스인 부분과 플러스인 부분을 heapq에 넣어서 이를 합치는 등의 최적화 과정을 수행하였지만,

결과적으론 그냥 sorted보다 더 나쁜 runtime을 가지게 되었다. 이에 대해서 설명한 글을 첨부한다.

[링크](https://leetcode.com/explore/challenge/card/december-leetcoding-challenge/571/week-3-december-15th-december-21st/3567/discuss/233054/Python-sorted()-solution-is-O(N)-not-O(NlogN))

![DISCUSSION](/images/2020-12-18-11-50-02.png)

즉 Python의 builtin sort는 timsort로, 이 경우엔 배열의 부분배열이 ascending order를 갖는지, decending order을 갖는지를 감지하고 이를 exploit하기 때문에 O(n)의 시간복잡도를 가진다는 것이다.

결국 내가 한 exploit 과정이 timsort에선 이미 구현되어 있었고, timsort가 더 좋았던 이유는 추가적인 최적화 + 라이브러리가 C여서가 아닐까?
