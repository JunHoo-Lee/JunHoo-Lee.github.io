주어진 Tree가 Balanced인지 판별하라

## Balanced Binary Tree

![Question](/images/Balanced%20Binary%20Tree.png)

문제를 파악하는데 시간이 오래 걸렸다. balanced tree가 단순히 최고 노드 기준으로 min depth와 max depth를 판단하는거라고 생각했는데, 그게 아니라 모든 노드에 대해서 그렇게 하는거였다.

따라서 그냥 뇌빼고 recursive하게 짰는데, 다음과 같다

```python
class Solution:
    def Balanced(self,root, isBal = True, depth = 0):
        if isBal == False:
            return False, 0
        ldep = 0
        rdep = 0
        lisBal = True
        risBal = True
        if root.left:
            lisBal , ldep = self.Balanced(root.left)
        if root.right:
            risBal , rdep = self.Balanced(root.right)
        
        if lisBal == False or risBal == False:
            return False, 0
        
        
        if abs(ldep - rdep) > 1:
            return False, 0
        return True, max(ldep,rdep) + 1

    def isBalanced(self, root: TreeNode) -> bool:
        if root == None:
            return True
        ret, _ = self.Balanced(root)
        return ret
```

즉 매 노드마다 length를 불러서 비교한 후, 위 노드로 length를 리턴하게 만드는 방식이다.

성능은 상위 30%정도 나왔고, 상위 90% 이상 나온 사람들의 코드도 확인해보았지만 방식은 같았지만, 최적화 측면에서 훨씬 나아 이를 소개한다.

```python
class Solution(object):
    def isBalanced(self, root):
        def check(root):
            if root is None:
                return 0
            left  = check(root.left)
            right = check(root.right)
            if left == -1 or right == -1 or abs(left - right) > 1:
                return -1
            return 1 + max(left, right)
            
        return check(root) != -1
```

우선 함수안에 함수넣는거 생각 안하고 클래스 내에 다른 함수로 생성하는데 그 습관부터 고쳐야 할 듯하고,

나같은 경우는 무조건 node가 들어온다고 생각하고 recursive하게 짜서 조금 noisy한데, 여기선 None도 처리해줘서 길이면에서 훨씬 짧아졌다.
